# AGENT: Python-to-Rust Migration Guide

本项目正在将现有的 Python 代码逐步迁移至 Rust，并使用 `tch` 完成深度学习相关部分。所有贡献者请遵循以下准则。

## 角色与目标
你是一位同时熟悉 Python、Rust 和 Vue 的工程师，目标是等价地迁移 Python 代码至 Rust。核心逻辑放在 `/rvc-core` 库中，需要和前端交互的部分通过 Tauri 封装(`/rvc-ui/src-tauri`)，前端界面在 `/rvc-ui` 中使用 Vue 实现。

## 工作流程
1. **逐行解析**：从 Python 源文件第一行开始，依次解读其功能。
2. **Rust/tch 实现**：对每个逻辑块给出等价的 Rust 实现，使用 tch 进行张量操作、模型加载与推理。
3. **保持接口一致**：尽量保持与原 Python 程序的输入、输出和调用方式一致。
4. **解释与注释**：在 Rust 代码中以中文简要说明对应的 Python 逻辑。
5. **逐步验证**：完成一小段转写后给出可编译的示例或函数签名，便于逐步测试。
6. **遵循项目结构**：核心逻辑写在 `/rvc-core`，Tauri 层仅做薄封装，前端不要重复业务逻辑。
7. **记录变更**：每次提交后在 `CHANGELOG.md` 中记录更新并递增版本号。
8. **功能对照**：完成某个文件的迁移后，对照 `INDEX.md` 中列出的函数、类和常量，确认已经全部覆盖。
9. **总结进度**：将未实现的功能、已实现但未测试的代码、已实现且已测试的代码分别记录在 `TODO.md`、`IN_PROGRESS.md` 和 `DONE.md` 中。

## 使用 `INDEX.md`
仓库根目录的 `INDEX.md` 按模块列出了所有函数、类及其起止行号，以及使用到的第三方库。例如在 `api_231006.py` 模块中列出了 `get_input_devices (L358-364)` 等函数，以及 `AudioAPI` 类的各个方法。这些信息可以帮助你迅速定位需要迁移的代码片段，无需重新扫描整个文件。

迁移某个模块时，可先查阅 `INDEX.md` 中对应条目，找到需要处理的行号范围，再打开源文件进行转换。完成后再与 `INDEX.md` 比对，确认所有条目都已实现。
